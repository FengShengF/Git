git 学习笔记

Git是分布式版本控制系统

Git跟svn的区别

Git是分布式，本地有镜像,无网络时也可以提交到本地镜像,待到有网络时再push到服务器
svn:非分布式.无网络不可以提交



知识点

生成秘钥：ssh-keygen -t rsa -C "git的账号邮箱" git diff -----比较工作区和暂存区的区别
git diff --cached -----比较暂存区和版本库的区别
git diff HEAD -- filename -----比较工作区和版本库的区别
git status-----掌握工作区的状态

git log-----查看提交的历史记录(多行显示),以便确定要回退span>到哪个版本
git log --pretty=oneline-----查看提交的历史记录(一行显示)

git reflog -----查看命令历史,以便确定要回到未来的哪个版本

git reset HEAD 文件名 -------丢弃对暂存区的修改

git reset --hard HEAD^-----回退到上一个版本
git reset --hard HEAD^^-----回退到上上个版本
git reset --hard 版本的id-----回退或者回到未来某个版本(可以用git reglog查看版本的id)

git checkout -- 文件名 --------丢弃对工作区的修改,回到最近一次git commit或git add时的状态 git checkout -b 本地分支名 origin/远程分支名----- 将远程git仓库里的指定分支拉取到本地
git push origin HEAD --force #远程提交回退
git branch-----查看所有分支
git branch 分支名 、git switch 分支名-----创建分之
git checkout 分支名-----切换分之
git checkout -b 分支名 、git switch -c 分支名-----创建并且切换分支
git merge 分支名-----合并某分之到当前分之
git branch -d 分支名-----删除分之

删除远程分之(远程分支存在的情况)

  • 1.git push origin --delete 分支名 或者 git push origin :分支名

    • 删除远程分之名(远程分支不存在的情况)

  • git remote prune 分支名(不好使？) --- git remote prune origin

    • 修改分之名称

  • 1.git branch -m 旧的分支名 新的分支名

    • 关联本地仓库和远端仓库的方法

    1.先有本地库，后有远程库的时候
     一.git init
     二.git add .
     三.git commit -m '想说的'
     四.git remote add origin git@github.com:名字/learngit.git(将本地库与远程库进行关联)
     五.git push -u origin master(-u 参数表示把本地的master分支和远程的master分支关联起来)

    失败后，1.git pull --rebase origin master 2.git push origin master
